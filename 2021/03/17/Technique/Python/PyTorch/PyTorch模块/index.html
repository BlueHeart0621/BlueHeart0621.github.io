PyTorch模块

1. 简介

Torch 模块主要包括 16 个主要模块和 6 个辅助模块。

2. 主要模块

2.1 torch 模块

torch 模块本身包含了 Torch 经常使用的一些激活函数，比如：

  • torch.sigmoid
  • torch.relu
  • torch.tanh

以及一些 Torch 张量生成函数，比如：

  • torch.zeros
  • torch.ones
  • torch.randn

此外，还包含一些针对 Torch 张量的操作，比如：

  • torch.mm（矩阵乘法）
  • torch.select（张量元素选择）

2.2 torch.Tensor 模块

torch.Tensor 模块定义了 Torch 中的张量的维数、形状和数值类型，比如：

  • torch.uint8
  • torch.int32
  • torch.float32
  • torch.LongTensor
  • torch.FloatTensor

同时，张量的类中也包含着一系列的方法，这些方法返回新的张量或更改当前的张量。按照 Torch 的命名规则，如果张量的某个类方法会返回张量：

  • 若方法后缀带下划线，则方法会修改张量本身的数据。
  • 若方法后缀不带下划线，则方法会返回新的张量。

torch.Storage 模块负责 torch.Tensor 模块底层的数据存储，即为一个张量分配连续的一维内存空间。

2.3 torch.sparse 模块

torch.sparse 模块定义了稀疏张量，其稀疏张量存储格式采用的是 COO 格式，稀疏张量之间可以做加减乘除和矩阵乘法。

2.4 torch.cuda 模块

torch.cuda 模块定义了与 cuda 运算的一系列函数和类，诸如：

  • torch.cuda.is_available()：检查系数的 cuda 是否可用
  • torch.cuda.device_count()：统计可访问到的 gpu 数量

2.5 torch.nn 模块

torch.nn 模块是 Torch 神经网络模块化的核心，这个模块下包括很多字模块，诸如：

  • torch.nn.ConvNd（卷积层）
  • torch.nn.Linear（线性层）
  • torch.nn.Pooling（池化层）

构建深度学习模型时，可以通过继承 nn.Module 类自定义神经网络。此外，torch.nn 模块中也定义了一系列的损失函数，诸如：

  • torch.nn.MSELoss
  • torch.nn.CrossEntropyLoss

2.6 torch.nn.functional 模块

torch.nn.functional 模块定义了一些与神经网络相关的函数，比如卷积函数和池化函数等。torch.nn 中定义的模块一般会调用 torch.nn.functional 中的函数。此外，torch.nn.functional 中还定义了一些不常用的激活函数，诸如：

  • torch.nn.functional.relu6
  • torch.nn.functional.elu

2.7 torch.nn.init 模块

torch.nn.init 模块定义了神经网络权重的初始化，诸如：

  • torch.nn.init.uniform、torch.nn.init.uniform_
  • torch.nn.init.normal、torch.nn.init.normal_

【注】以下划线结尾的方法会直接修改原张量，非下划线结尾则是返回新张量。

2.8 torch.optim 模块

torch.optim 模块定义了一系列的优化器，诸如：

  • torch.optim.SGD
  • torch.optim.AdaGrad
  • torch.optim.RMSProp
  • torch.optim.Adam

此外，还包含学习率衰减的算法模块 torch.optim.lr_scheduler，该模块包含了一些学习率下降算法，诸如：

  • torch.optim.lr_scheduler.StepLR
  • torch.optim.lr_scheduler.CosineAnnealingLR

2.9 torch.autograd 模块

torch.autograd 模块是 Torch 的自动微分算法模块，该模块定义了一系列自动微分函数，诸如：

  • torch.autograd.backward：用于在求得损失函数后进行反向梯度传播。
  • torch.autograd.grad：用于一个标量张量对另一个张量求导，以及在代码中设置不参与求导的部分。

此外，该模块还内置了数值梯度功能和检查自动微分引擎是否输出正确结果的功能。

2.10 torch.distributed 模块

torch.distributed 模块是 Torch 的分布式计算模块，主要功能是提供 Torch 的并行运行环境，其主要支持的后端有 MPI、Gloo、NCCL 三种。

  • Torch 的分布式工作原理主要是启动多个并行的进程，每个进程都拥有一个模型的备份，然后输入不同的训练数据到多个并行的进程，计算损失函数，每个进程独立地做反向传播，最后对所有进程权重张量的梯度做规约。
  • 用到后端的部分主要是数据的广播和数据的收集。前者是把数据从一个节点（进程）传播到另一个节点（进程），比如规约后的梯度张量的传播；后者则是把数据从其他节点（进程）转移到当前节点（进程），比如把梯度张量从其他节点转移到某个特定的节点，然后对所有的张量求平均。
  • Torch 的分布式计算模块不但提供了后端的一个包装，还提供了一些启动方式来启动多个进程，包括但不限于通过网络（TCP）、环境变量、共享文件等。

2.11 torch.distributions 模块

torch.distributions 提供了一系列类，使得 Torch 能够对不同的分布进行采样，并且生成概率采样过程的计算图。诸如：

  • torch.distributions.Categorical
  • torch.distributinos.Normal

在一些应用中，比如强化学习，经常会使用一个深度学习模型来模拟在不同环境条件下采取的策略，其最后的输出是不同动作的概率。当深度学习模型输出概率后，需要根据概率对策略进行采样来模拟当前的策略概率分布，最后用梯度下降方法来让最优策略的概率最大（这个算法被称为「策略梯度算法」，）。实际上，由于采样的输出结果是离散的，无法直接求导，所以不能使用反向传播算法。

2.12 torch.hub 模块

torch.hub 模块提供了一系列预训练的模型供用户使用。

  • torch.hub.list 函数用于获取某个模型镜像站点的模型信息。
  • torch.hub.load 函数用来载入预训练的模型，载入后的模型可以保存到本地。

此外，该模型还提供这些模型对应类支持的方法。

2.13 torch.jit 模块

torch.jit 模块是 Torch 的即时编译器模块，这个模块主要是把 Torch 的动态图转换成可以优化和序列化的静态图。其主要工作原理是：

  • 通过预先定义好的张量，追踪整个动态图的构建过程，得到最终构建出来的动态图，然后转换为静态图。

通过 JIT 得到的静态图可以被保存，并且被 Torch 其他前端支持（比如 C++ 的前端）。另外，JIT 也可以用来生成其他格式的神经网络描述文件，比如 ONNX。

torch.jit 模块支持两种模式，即「脚本模式（ScriptModule）」和「追踪模式（Tracing）」。二者都能构建静态图，区别在于前者支持控制流，后者不支持，但前者支持的神经网络模块比后者少。

2.14 torch.multiprocessing 模块

torch.multiprocessing 模块定义了 Torch 中的多进程 API，可以启动不同的进程，每个进程运行不同的深度模型，并且能够在进程间共享张量。共享的张量可以在 CPU 上，也可以在 GPU 上。多进程 API 还提供了与原生的多进程 API （即 multiprocessing 库）相同的一系列函数，诸如「锁」和「队列」。

2.15 torch.random 模块

torch.random 模块提供一系列方法来保存和设置随机数生成器的状态，诸如：

  • get_rng_state：获取当前随机数生成器的状态
  • set_rng_state：设置当前随机数生成器的状态
  • manual_seed：设置随机种子
  • initial_seed：得到程序初始的随机种子

因为神经网络训练是一个随机的过程，包括数据的输入、权重的初始化都具有一定的随机性，设置一个统一的随机种子可以有效地帮助我们测试不同神经网络的表现，有助于调试神经网络结构。

2.16 torch.onnx 模块

torch.onnx 模块定义了 Torch 导出和载入 ONNX 格式的深度学习模型描述文件。ONNX 格式的存在是为了方便不同的深度学习框间之间交换模型，引入这个模块可以方便 Torch 导出模型给其他深度学习框架使用，同时让 Torch 更方便地载入其他深度学习框架导出的模型来构建深度学习模型。

2.17 torch.utils 模块

torch.utils 模块提供了一系列工具用来帮助神经网络训练、测试和结构优化。这个模块主要包含 6 个子模块：

2.17.1 torch.utils.bottleneck 模块

torch.utils.bottleneck 模块用来检查深度学习模型中模块的运行时间，从而可以找到性能瓶颈的那些模块，通过优化那些模块的运行时间，从而优化整个深度学习模型的性能。

2.17.2 torch.utils.checkpoint 模块

torch.utils.checkpoint 模块用来节约深度学习使用的内存。

  • Torch 进行梯度反向传播时，在构建计算图的时候需要保存中间数据，而这些数据大大增加了深度学习的内存消耗。为了减少内存消耗，让迷你批次的大小得到提高，从而提升深度学习模型的性能和优化时的稳定性，可以通过这个模块记录中间数据的计算过程，然后丢弃这些中间数据，等需要用到时再重新计算这些数据。

这个模块设计的核心思想就是以计算时间换内存空间。

2.17.3 torch.utils.cpp_extension 模块

torch.utils.cpp_extension 模块定义了 Torch 的 C++ 扩展，其主要包含两个类：

  • CppExtension：定义了使用 C++ 来编写的扩展模块的源代码相关信息。
  • CUDAExtension：定义了 C++/CUDA 编写的扩展模块的源代码相关信息。

在某些情况下，用户可能使用 C++ 实现某些张量运算和神经网络结构（比如 Torch 中没有类似功能的模块或者类似功能的模块性能较低），就可以使用该模块来调用用户自己用 C++/CUDA 编写的绳度学习扩展模块。在底层上，这个扩展模块使用了 pybind11，保持了接口的轻量性并使得 Torch 易于被扩展。

2.17.4 torch.utils.data 模块

torch.utils.data 模块引入了数据集（Dataset）和数据载入器（DataLoader）的概念，前者代表包含了所有数据的数据集，通过索引能够得到某一条特定的数据，后者通过对数据集的包装，可以对数据集进行随机排列和采样，得到一系列打乱数据的迷你批次。

2.17.5 torch.utils.dlpacl 模块

torch.utils.dlpacl 模块定义了 Torch 张量和 DLPackz 张量存储格式之间的转换，用于不同框架之间张量数据的交换。

2.17.6 torch.utils.tensorboard 模块

torch.utils.tensorboard 模块是对 TensorBoard 数据可视化工具的支持。

  • TensorBoard 原来是 TensorFlow 自带的数据可视化工具，能够显示深度学习模型在训练过程中损失函数、张量权重的直方图，以及模型训练过程中输出的文本、图像和视频等。
  • TensorFlow 功能非常强大，而且是基于可交互的动态网页设计的，使用者可以通过预先提供的一系列功能来输出特定的训练过程的细节。

Torch 支持 TensorBoard 可视化后，在训练过程中，可以很方便地观察中间变量输出的张量，从而有利于调试深度学习模型。
