  • 注意 view() 返回的新 tensor 与源 tensor 共享内存（其实是同⼀个 tensor ），也即更改其中的⼀个，另外⼀个也会跟着改变。(顾名思义，view 仅仅是改变了对这个张量的观察⻆度)

1. Numpy 与 Tensor 相互转换

1.1 共享内存

⽤ numpy() 和 from_numpy() 将 Tensor 和 NumPy 中的数组相互转换。

1.2 非共享内存

用 torch.tensor() 将 NumPy 数组转换成 Tensor ，需要注意的是该方法总是会进行数据拷贝，返回的 Tensor 和原来的数据不再共享内存。

2. 模块

2.1 nn 模块

torch.nn 模块提供大量的神经网络的层，以及各种损失函数等。

  • torch.nn 的核心数据结构是 Module ，它是⼀一个抽象概念，既可以表示神经网络中的某个层（layer），也可以表示⼀一个包含很多层的神经网络。

  • torch.nn.init 模块定义了各种初始化方法。

2.2 optim 模块

torch.optim 模块提供学习的优化算法。

2.3 utils 模块

  • torch.utils.data 模块定义了有关数据处理的工具。

3. Module 类

Module 类是 nn 模块⾥提供的⼀个模型构造类，是所有神经⽹络模块的基类，我们可以继承它来定义我们想要的模型。

  • Sequential 、 ModuleList 、 ModuleDict 类都继承⾃ Module 类。

3.1 Sequential 类

对于 Sequential 实例中含模型参数的层，我们可以通过 Module 类的 parameters() 或者 named_parameters ⽅法来访问所有参数（以迭代器的形式返回），后者除了返回参数 Tensor 外还会返回其名字。

  • 对于使⽤ Sequential 类构造的神经⽹络，我们可以通过⽅括号 [] 来访问⽹络的任⼀层。

  • 使用 parameters() 访问多层网络所有的参数返回的是参数 Tensor；而使用 parameter() 访问单层网络的所有参数返回的是参数 torch.nn.parameter.Parameter，这是 Tensor 的子类，二者的不同在于：如果⼀
    个 Tensor 是 Parameter ，那么它会⾃动被添加到模型的参数列表⾥。

4. CPU & GPU

默认情况下， Tensor 会被存在内存上。因此，之前我们每次打印 Tensor 的时候看不到GPU相关标识。

  • 通过 Tensor 的 device 属性来查看该 Tensor 所在的设备。

4.1 CPU to GPU

  • 使⽤ .cuda() 可以将CPU上的 Tensor 转换（复制）到GPU上。如果有多块GPU，我们⽤ .cuda(i) 来表示第 块GPU及相应的显存（ 从0开始）且 cuda(0) 和 cuda() 等价。

  • 如果对在GPU上的数据进⾏运算，那么结果还是存放在GPU上。

  • 存储在不同位置中的数据是不可以直接进⾏计算的。即存放在CPU上的数据不可以直接与存放在GPU上的数据进⾏运算，位于不同GPU上的数据也是不能直接进⾏计算的。

要想在 GPU 上训练模型，需要保证模型的输入的 Tensor 和模型都在同一设备上。
